Deployment: Access & Reproduction

Acme Logistics inbound carrier sales — FastAPI backend, SQLite, Chart.js operations dashboard. Production is hosted on Railway; source: github.com/piercegf/happy-robot-fde.

1. How to access the deployment

Production base URL: https://happy-robot-fde-production-f148.up.railway.app

Resource URL / method
Operations dashboard https://happy-robot-fde-production-f148.up.railway.app/dashboard — open in a browser. The server injects the API_KEY from Railway into the page. Do not open dashboard/index.html as a static file.
API root https://happy-robot-fde-production-f148.up.railway.app/ redirects to /dashboard.
Health check (no API key) GET https://happy-robot-fde-production-f148.up.railway.app/health — JSON: status, timestamp, version.
Authenticated API Every route except /, /health, and GET /dashboard requires X-API-Key equal to the API_KEY environment variable on the server (Railway → service → Variables). Examples: GET /api/metrics, POST /api/loads/search, POST /api/calls/log.

HTTPS: Railway terminates TLS; production traffic uses https:// only.

Admin surface: Anyone who can open /dashboard can view the injected key in browser devtools. Use a strong API_KEY in Railway and rotate it if the URL is shared widely.

HappyRobot workflow: Set HTTP tool and webhook base URL to https://happy-robot-fde-production-f148.up.railway.app (no trailing slash on the host; paths are /api/... as configured in tools). Workflow: https://platform.happyrobot.ai/fdealejandroperez/workflows/gchtmr5tol1e.

2. How to reproduce the deployment

Infrastructure is not managed with Terraform. Use Railway, Docker Compose, or local Python as below.

Option A — Railway (current production)

  1. Prerequisites: GitHub account, Railway account, access to piercegf/happy-robot-fde.
  2. Create service: Railway → New Project → Deploy from GitHub → choose piercegf/happy-robot-fde (default branch main). Build uses railway.json (Dockerfile builder) and the repo Dockerfile.
  3. Variables (Railway → service → Variables), minimum:
Variable Required Purpose
API_KEY Yes Authenticates API requests; injected into rendered /dashboard.
PORT No Set by Railway; app binds ${PORT:-8000}.
FMCSA_WEBKEY No Primary FMCSA mobile API; VerifyCarrier fallback still used when possible if unset.
DB_PATH No Default data/carrier_sales.db. Align with a mounted volume path if you add persistent storage.
  1. Persistence (optional): Volume mount e.g. /app/data and set DB_PATH accordingly so SQLite survives redeploys.
  2. Deploy: Push to the connected branch; Railway builds and deploys. This project’s live hostname is happy-robot-fde-production-f148.up.railway.app (yours may differ if you create a new service).

Start command (railway.json): uvicorn app.main:app --host 0.0.0.0 --port ${PORT:-8000}; restart on failure, up to 10 retries.

Option B — Docker Compose (local or any Docker host)

Clone https://github.com/piercegf/happy-robot-fde.git, then from repo root:

cd happy-robot-fde
cp .env.example .env
# .env.example already sets a local API_KEY; uncomment FMCSA_WEBKEY there if you use the primary FMCSA API
mkdir -p data
docker compose up --build

Dashboard: http://localhost:8000/dashboard. Compose binds ./data for SQLite persistence.

Option C — Local Python

git clone https://github.com/piercegf/happy-robot-fde.git
cd happy-robot-fde
python3 -m venv venv   # Python 3.12+ (matches Dockerfile)
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
mkdir -p data
uvicorn app.main:app --reload --port 8000

Run commands from the repo root so load_dotenv() finds .env. Windows: venv\Scripts\activate. Dashboard: http://localhost:8000/dashboard. API calls need X-API-Key matching API_KEY in .env (.env.example default acme-logistics-2026 for local dev).

Optional — Web voice QA (local only)

Run server/ (token API, port 3001) and client/ (Vite) per repository README. Production demos use HappyRobot web call; this stack is for local testing only.

3. Container summary

  • Image: python:3.12-slim; pip install -r requirements.txt; app at /app in the container.
  • Process: One Uvicorn worker (fine for demo load; scale by adding Railway replicas or multiple workers if needed later).
